## better-og とは？

**better-og** は、リクエスト元のプラットフォームに自動的に適応するOpen Graphイメージを生成するためのユーティリティセットです。Twitter、Telegram、Slack、iMessage、Instagramのいずれでリンクが共有されても、better-ogは呼び出し元を検出し、最適なアスペクト比でイメージをレンダリングします。

### 主な機能

- **アスペクト比の自動検出** -- リクエスト元のプラットフォーム（Twitter、Telegram、Slack、iMessage、Instagram）を検出し、適切なサイズでレンダリングします。
- **ロケール対応フォントフォールバック** -- Google Fontsの統合により、`en`、`ja`、`ko`、`zh`、`ar`、`hi`、`ru`、`uk` の自動フォールバックフォントを提供します。
- **プラットフォームセーフエリア** -- Twitterの下部クロップゾーンなど、プラットフォーム固有のセーフエリアインセットを提供します。
- **マルチランタイムアダプター** -- Next.js（Node & Edge）、Cloudflare Workers、TanStack Startに対応します。
- **2つのレンダリングプロバイダー** -- `next/og`（Vercel内蔵のSatori）または [Takumi](https://takumi.kane.tw/docs/)（WASMベースレンダリング）から選択できます。

### アスペクト比プリセット

| プリセット  | サイズ      | 比率     | 使用プラットフォーム |
| ----------- | ----------- | -------- | -------------------- |
| `STANDARD`  | 1200 x 630  | 1.91 : 1 | Twitter、汎用        |
| `SQUARE`    | 1200 x 1200 | 1 : 1    | Telegram、Slack      |
| `PORTRAIT`  | 630 x 1200  | 1 : 1.91 | iMessage             |
| `INSTAGRAM` | 1200 x 1500 | 4 : 5    | Instagram            |

### 動作の仕組み

1. ミドルウェアがOGイメージリクエストをインターセプトし、`User-Agent` ヘッダーを検査します。
2. 検出されたプラットフォームに基づいて、`aspect_ratio` クエリパラメータがURLリライトで追加されます。
3. OGルートハンドラーがアスペクト比を読み取り、サイズとセーフエリアを解決し、ロケール対応フォントをロードして、正しいサイズでコンポーネントをレンダリングします。

## インストール

必要なランタイムパッケージのみをインストールしてください。

```bash tab="Next.js (next/og)"
pnpm add @better-og/core next react
```

```bash tab="Next.js + Takumi"
pnpm add @better-og/core @takumi-rs/image-response next react
```

```bash tab="Edge / Workers"
pnpm add @better-og/core @takumi-rs/image-response @takumi-rs/wasm next react
```

```bash tab="TanStack Start"
pnpm add @better-og/core @takumi-rs/image-response react
```

## 最初のルート（Next.js）

Next.js アプリで最初のOGルートを作成します。

```tsx title="app/og/[lang]/route.tsx"
import { resolveFontSetup } from "@better-og/core";
import {
  createOgRouteHandler,
  loadGoogleFontForImageResponse,
} from "@better-og/next";

export const runtime = "nodejs";
export const revalidate = false;

const fontSetup = await resolveFontSetup({
  fonts: await loadGoogleFontForImageResponse({
    family: "Geist",
    weights: [400, 700],
  }),
  fallbackFontLocales: ["ja"],
});

export const GET = createOgRouteHandler({
  component: (ogContext) => (
    <div
      style={{
        display: "flex",
        width: "100%",
        height: "100%",
        alignItems: "center",
        justifyContent: "center",
        fontFamily: fontSetup.families.base,
        paddingBottom: 32 + ogContext.safeArea.bottom,
        background: "linear-gradient(135deg, #0d192c, #174a70, #4993d6)",
        color: "white",
      }}
    >
      <div style={{ fontSize: 64, fontWeight: 700 }}>better-og へようこそ</div>
    </div>
  ),
  fonts: fontSetup.fonts,
});
```

## プロキシの追加

自動アスペクト比検出を有効にするには、プロキシを追加します。

```ts title="proxy.ts"
import { withOgRewrite } from "@better-og/next";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  return withOgRewrite(request);
}

export const config = {
  matcher: ["/og/:path*"],
};
```

## Next.js 設定

Takumi プロバイダーを使用する場合は、外部パッケージとして設定します。

```ts title="next.config.ts"
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  serverExternalPackages: ["@takumi-rs/image-response"],
};

export default nextConfig;
```
