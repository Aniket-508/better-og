## Available Adapters

better-og provides dedicated adapters for different runtimes. Each adapter wraps the core logic and exposes a `createOgRouteHandler` (or equivalent) tailored to its environment.

| Import                     | Runtime              | Provider                            |
| -------------------------- | -------------------- | ----------------------------------- |
| `better-og/next`           | Next.js (Node)       | `next/og` or Takumi                 |
| `better-og/next/edge`      | Next.js (Edge)       | `next/og` only                      |
| `better-og/edge`           | Generic WASM runtime | Takumi (requires explicit `module`) |
| `better-og/workers`        | Cloudflare Workers   | Takumi (bundled WASM module)        |
| `better-og/tanstack-start` | TanStack Start       | Takumi                              |

## Choosing an Adapter

- **Next.js apps**: Use `better-og/next` for the Node runtime (supports both `next/og` and Takumi providers) or `better-og/next/edge` for the Edge runtime (uses `next/og`).
- **Cloudflare Workers**: Use `better-og/workers` for a preconfigured Takumi WASM setup.
- **Other edge runtimes**: Use `better-og/edge` with your own Takumi WASM module.
- **TanStack Start**: Use `better-og/tanstack-start` for server-side rendering with Takumi.

## Common Options

All adapters accept the same base options from `OgAdapterOptions`:

```ts
interface OgAdapterOptions {
  getOgContext?: (req: Request) => OgContext | Promise<OgContext>;
  localeFromRequest?: (req: Request) => string | undefined;
  format?: "webp" | "png";
  loadDefaultFonts?: boolean;
  fonts?: Font[];
  getFontsForLocale?: (locale: string) => Font[] | Promise<Font[]>;
  getFallbackFontsForLocale?: (locale: string) => Font[] | Promise<Font[]>;
  fallbackFontLocales?: string[];
}
```
